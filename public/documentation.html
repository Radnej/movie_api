MyMovies Application

This is the server-side component of a “movies” web application.
The web application will provide users with access to information about different movies,
directors, and genres. Users will be able to sign up, update their personal information,
and create a list of their favorite movies.

Business Logic URL HTTP Method Query parameters Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None None Returns JSON Object with a list of all top movies 
          
        Title: String,
        Director: String,
        Stars: [String],
        Genre: {
          Name: String,
          Description: String
        },
        Director: {
          Name: String,
          Bio: String
        }
        Featured: Boolean,
        ImageUrl: String
      }
Return data about a single movie by name to the user /movies/[title] GET :title None A JSON object holding data about the selected movie containing description, genre, director and img URL. Example: 
          
            {
              Title: String,
              Director: String,
              Stars: [String],
              Genre: {
                Name: String,
                Description: String
              },
              Director: {
                Name: String,

              }

              ImageUrl: String
            }
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/geners/[name] GET :genre None A JSON object holding data about the selected genre. Example: 
          
{ 
  name: 'Comedy', 
  description: '', 
}
Return data about a director (bio, birth year) by name /movies/directors/[name] GET :name None A JSON object holding data about the selected director. Example: 
          
{ 
  name: String 
  birth: String 
}
Allow new users to register /users POST None A JSON object holding data about the user to add. Example: 
          
{
  Username: String,
  (required)
  Password: String,
  (required)
  Email: String,
  (required)
  Birthday: Date
}
Allow users to update their user info (username) /users/[username] PUT :username A JSON object holding the new username. Example: 
          
{ 
  "username": "Mary"
}
A JSON object with the updated user profile. Example: 
          
            {
              Username: String,
              (required)
              Password: String,
              Email: String,
              Birthday: Date
            }
Allow users to add a movie to their list of favorites /users/[username]/[movie] POST :username, :movie None A text message indicating the movie was added
Allow users to remove a movie from their list of favorites /users/[username]/[movie] DELETE :username, :movie None A text message indicating the movie was deleted
Allow existing users to deregister /users/[username] DELETE :username None A message indicating the user was deregistered
